## Update Rules

1. 코드 구조가 바뀌면 `02 시스템 아키텍처`와 `03 ROS 토픽 및 통신 설계`를 확인한다.
2. 대회 규정이나 MORAI 가이드에서 새 제약을 발견하면 `01 대회 규정 요약`을 갱신한다.
3. 새 기능 구현, 실험, 실패 분석은 `04 개발 로그`에 누적한다.
4. 실제 주행 실험은 `05 실험 로그 템플릿` 형식을 복제해서 날짜별로 기록한다.
5. 큰 방향이 바뀌면 `00 프로젝트 마스터 플랜`에 반영한다.
